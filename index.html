🛠️ Develop API

商户开发接口，用于配方管理、制作记录查询等业务场景。

Base URL https://mis-api.bingjoy.com.cn
版本 openapi_3_1
规范 OAS 3.0
🔐 认证方式：请求头添加 Authorization: X-API-Key {yourApiKey}

1. Recipe 配方管理

商户配方管理接口（Merchant recipe management interface）

POST /api/recipes 创建配方 / Create recipe
请求体 (application/json)
JSON{
  "code": "R001",
  "externalCode": "EXT001",
  "name": "拿铁咖啡",
  "type": "咖啡",
  "remark": "经典配方",
  "materials": ["material-id-1", "material-id-2"],
  "specifications": ["spec-id-1", "spec-id-2"]
}

字段说明见 RecipeCo

响应 (200 OK)
JSON{
  "errcode": 0,
  "errmsg": "success",
  "data": { /* Recipe 对象，见数据模型 */ },
  "timestamp": 1715040000000
}
GET /api/recipes/{id} 配方详情 / Recipe detail

2. Recipe Excel 配方导入导出

通过 Excel 文件批量导入配方、下载导出配方数据

POST /api/recipes/fromExcel 用 Excel 创建配方 / Create recipes from Excel
请求体 (multipart/form-data)
字段类型必填说明
filebinary✅ 必填Excel 文件
cURL 示例
bashcurl -X POST "https://mis-api.bingjoy.com.cn/api/recipes/fromExcel" \
  -H "Authorization: X-API-Key {apiKey}" \
  -H "merchantId: your-merchant-id" \
  -F "file=@recipes.xlsx"
响应 (200 OK)
JSON{
  "errcode": 0,
  "errmsg": "success",
  "data": null,
  "timestamp": 1715040000000
}
POST /api/recipes/download 下载选中项配方 / Download the recipe for the selected item
请求体 (application/json)
JSON{
  "recipeId": ["recipe-id-1", "recipe-id-2"]
}
字段类型必填说明
recipeIdstring[]✅ 必填要下载的配方 ID 列表
响应 (200 OK)
JSON{
  "errcode": 0,
  "errmsg": "success",
  "data": null,
  "timestamp": 1715040000000
}

4. 数据模型

API 涉及的所有数据结构定义

Recipe 配方
配方完整数据对象，包含规格、物料、配方项等关联信息
字段类型说明
idstring配方 ID
namestring名称 / Name
codestring编码 / Code
externalCodestring外部编码 / External Code
typestring分类 / Type
statusinteger(int32)状态：-1 已删除 / 1 正常 / 0 禁用
remarkstring备注
timestampinteger(int64)创建时间戳
specificationsSpecification[]规格列表
itemItem[]配方项列表
materialsMaterial[]物料列表
invalidateMaterialsBillMaterial[]失效物料列表
RecipeCo 创建/更新配方请求
用于创建和更新配方的请求体结构
字段类型必填说明
codestring❌ 可选配方编码
externalCodestring❌ 可选外部编码
namestring❌ 可选配方名称
typestring❌ 可选配方分类
remarkstring❌ 可选备注
materialsstring[]❌ 可选物料 ID 列表
specificationsstring[]❌ 可选规格 ID 列表
RecipePageRequest 配方分页请求
配方列表分页查询参数
字段类型必填说明
pageinteger(int32)✅ 必填页码
limitinteger(int32)✅ 必填每页条数
sidxstring❌ 可选排序字段
sordstring❌ 可选排序方向
keywordstring❌ 可选搜索关键词
statusinteger(int32)❌ 可选状态筛选
categorystring❌ 可选分类筛选
StatusCo 状态变更请求
更改配方状态时的请求体
字段类型必填说明
statusinteger(int32)✅ 必填1 = 正常 Normal，0 = 禁用 Disable
Batch 批量删除请求
批量删除配方时的请求体
字段类型必填说明
idsstring[]❌ 可选要删除的 ID 列表
Download 下载请求
下载选中项配方时的请求体
字段类型必填说明
recipeIdstring[]❌ 可选要下载的配方 ID 列表
ProductionRecordPage 制作记录分页请求
制作记录分页查询参数
字段类型必填说明
pageinteger(int32)✅ 必填页码
limitinteger(int32)✅ 必填每页条数
sidxstring❌ 可选排序字段
sordstring❌ 可选排序方向
keywordstring❌ 可选搜索关键词
statusinteger(int32)❌ 可选状态筛选
startDateinteger(int64)❌ 可选起始日期（时间戳，毫秒）
endDateinteger(int64)❌ 可选结束日期（时间戳，毫秒）
Specification 规格
配方规格定义，如温度、杯型等
字段类型说明
idstring规格 ID
namestring名称（如：温度、杯型）
codestring编码
statusinteger(int32)1 正常 / 0 禁用
timestampinteger(int64)创建时间戳
valuesSpecificationValue[]规格值列表
SpecificationValue 规格值
规格的具体值，如常温、热、冰
字段类型说明
valuestring值（如：常温、热、冰）
codestring编码
externalCodestring外部编码
Item 配方项
配方的具体组成项，关联规格值和物料
字段类型说明
idstring配方项 ID
specificationValuesSpecificationValue[]物料规格值
materialsBillMaterial[]物料列表（含重量）
othermap<string, object>其他扩展属性
Material 物料
基础物料数据对象
字段类型说明
idstring物料 ID
namestring物料名称 / Material Name
codestring物料编码 / Material Code
statusinteger(int32)物料状态 / Material Status
groupstring物料分组 / Material Group
timestampinteger(int64)创建时间戳 / Timestamp
expirationDateinteger(int64)过期时间戳 / Expiration Date
BillMaterial 物料（含重量）
带重量信息的物料对象，用于配方项和失效物料
⬆ 继承 Material 所有字段
字段类型说明
valuenumber物料重量 / Material Weight
ApiResponse<T> 通用响应结构
所有接口均返回统一的响应结构
字段类型说明
errcodeinteger(int32)错误码，0 表示成功
errmsgstring错误信息，"success" 表示成功
dataT响应数据，类型视接口而定
timestampinteger(int64)响应时间戳（毫秒）

分页数据结构 Page<T>

字段类型说明
pageinteger(int32)当前页码
limitinteger(int32)每页条数
totalPagesinteger(int64)总页数
totalElementsinteger(int64)总记录数
contentT[]数据列表
